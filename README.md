### 서울시 흡연시설 지도 웹

#### 이슈
- GitHub Pages는 정적 사이트를 배포할 때 보통 배포 대상으로 지정한 위치에 index.html이 있어야 정상적으로 첫 화면을 띄우는데, 그 파일이 없어 링크가 떠도 404가 남. (현재 templetes 안에 위치)
- 또한 해당 프로젝트는 한때 Flask 서버 실행 방식을 전제로 개발되어, 그대로는 Pages에 바로 맞지 않음
  - 기존 서버 렌더링 구조 -> 정적 배포용으로 한 번 바꿔야 함
  - gpt에게 문의 중:
    - Pages용으로 살릴 수 있는지,
    - templates 안의 HTML을 루트용 index.html로 빼야 하는지,
    - Flask 의존 부분을 정적으로 바꿔야 하는지
